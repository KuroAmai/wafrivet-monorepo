# Backend — Email Integration Guide
**For the backend dev / co-founder**

---

## What We're Using

- **Resend** — email delivery service
- **React Email** — renders the templates to HTML
- The frontend team owns the templates. The backend just receives rendered HTML and sends it.

---

## Step 1 — Create a Resend Account

Go to `resend.com` and sign up.

Once in, go to **Domains → Add Domain** and type `wafrivet.com`.

Resend will give you a set of DNS records to add. They look like this:

```
Type    Name                          Value
TXT     resend._domainkey.wafrivet.com   [long string]
MX      send.wafrivet.com               feedback-smtp.us-east-1.amazonses.com
TXT     send.wafrivet.com               v=spf1 include:amazonses.com ~all
```

Go to **Cloudflare → wafrivet.com → DNS → Records** and add each one exactly as Resend shows them. Don't change anything. Once added, go back to Resend and hit Verify. It can take up to 15 minutes but usually happens in under 2.

Once verified the domain shows a green Verified badge in Resend.

---

## Step 2 — Get the API Key

In Resend go to **API Keys → Create API Key**.

Name it `wafrivet-production`.

Copy the key — it only shows once.

Add it to the backend `.env`:

```
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxx
FROM_EMAIL=no-reply@wafrivet.com
FROM_NAME=Wafrivet
```

---

## Step 3 — Install Dependencies

In the backend repo run:

```bash
npm install resend
```

That's the only package needed on the backend side.

---

## Step 4 — Create the Email Service File

Create this file:

```
src/services/email.service.js
```

This is the only file that touches Resend. Every other part of the backend calls functions from this file — nothing calls Resend directly.

The file has this structure:

```
1. Import Resend and initialise with API key
2. One send function per email type
3. Each function takes props, accepts HTML, and sends via Resend
```

The HTML comes from the frontend email package. The backend does not build HTML — it just receives it and sends it.

---

## Step 5 — How the Backend Receives Templates

The frontend team will publish the email package to the monorepo as `@wafrivet/email`. The backend imports from it like this:

```javascript
import { renderWelcomeEmail } from '@wafrivet/email'
import { renderOTPEmail } from '@wafrivet/email'
import { renderOrderConfirmationEmail } from '@wafrivet/email'
```

Each `render` function takes props and returns an HTML string. The backend passes that string to Resend.

Until the frontend package is ready, the backend dev can use placeholder HTML strings for testing — just a simple `<p>Test email</p>` — and swap in the real templates when they're ready.
